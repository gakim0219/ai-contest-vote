# HD현대 AI Agent Challenge 평가 시스템 기획서

## 1. 시스템 개요

HD현대 AI Agent 경진대회의 **사전심사 · 본선심사 · 인기투표 · 예측 · 시상 · 추첨**을 위한 웹 기반 평가 시스템.
7개 독립 페이지로 구성되며, 각 페이지는 별도 URL(QR코드)로 접근한다.

**기술 스택**: React 18 + Vite + Tailwind CSS + Firebase Hosting + Firestore

---

## 2. 페이지 구성 (7개 독립 URL)

| # | URL | 페이지명 | 대상 | 인증 |
|---|------|---------|------|------|
| 1 | `/preliminary` | 사전심사 투표 | 일반 직원 | 사번+이름 (1인1회) |
| 2 | `/vote` | 본선 인기투표 | 일반 직원 | 사번+이름 (1인1회) |
| 3 | `/judge` | 심사위원 평가 | 심사위원 | 이름 → 명단 검증 |
| 4 | `/predict` | 1등을 맞혀라 | 일반 직원 | 사번+이름 (1인1회) |
| 5 | `/ceremony` | 시상 화면 | 관리자 | 비밀번호 |
| 6 | `/admin` | 관리자 집계 | 관리자 | 비밀번호 |
| 7 | `/draw` | 추첨 화면 | 관리자 | 비밀번호 |

- 각 페이지는 **완전 독립** (탭/네비게이션 없음)
- QR코드 7개로 현장 배포

## 3. 각 페이지 상세

### 3-1. 사전심사 투표 (`/preliminary`)

**대상**: 일반 직원
**인증**: 사번(A000000 형태) + 이름 → 동일 사번 중복 차단
**액센트 색상**: Orange `#f97316`

**플로우**:
```
사번+이름 입력 → 사번 중복 체크
  → 이미 투표 → "이미 투표하셨습니다"
  → 미투표 → 팀 목록 → 1팀 선택 → 투표 완료
```

---

### 3-2. 본선 인기투표 (`/vote`)

**대상**: 일반 직원
**인증**: 사번(A000000 형태) + 이름 → 동일 사번 중복 차단 (명부 대조 없음)
**액센트 색상**: Indigo `#6366f1`

**플로우**:
```
사번+이름 입력 → 사번 중복 체크
  → 이미 투표 → "이미 투표하셨습니다"
  → 미투표 → 팀 목록 → 1팀 선택 → 투표 완료
```

- 경고: "1인 1회 / 부정 투표 시 무효"

---

### 3-3. 심사위원 평가 (`/judge`)

**대상**: 관리자 페이지에서 등록된 심사위원만
**인증**: 이름 입력 → Firestore `judges` 컬렉션 대조
**액센트 색상**: Amber `#f59e0b`

**평가 항목**:

| 항목 | 설명 | 기본 가중치 |
|------|------|------------|
| 혁신성 | AI Agent 활용의 창의성과 독창성 | 25% |
| 완성도 | 프로토타입의 기능 구현 수준 | 25% |
| 업무 영향도 | 실제 업무 효율화 기여 가능성 | 30% |
| 발표력 | 발표 구성력과 전달력 | 20% |

- 각 항목 1~10점 (별점 UI)
- 팀별 탭으로 전환하며 평가
- 모든 팀 × 모든 항목 완료 시 제출 가능
- 재접속 시 기존 점수 로드 (upsert)

---

### 3-4. 1등을 맞혀라 (`/predict`)

**대상**: 일반 직원
**인증**: 사번+이름 → 동일 사번 중복 차단
**액센트 색상**: Pink `#ec4899`

**플로우**:
```
사번+이름 입력 → 사번 중복 체크
  → 이미 예측 → "이미 참여하셨습니다"
  → 미참여 → "최종 1위가 될 팀" 1개 선택 → 제출 완료
```

- /vote와 별개 (같은 사번이 양쪽 참여 가능)
- "맞히면 추첨을 통해 경품 증정!"

---

### 3-5. 시상 화면 (`/ceremony`)

**대상**: 관리자 (대형 스크린 송출용)
**인증**: 관리자 비밀번호
**액센트 색상**: Gold `#facc15`

**플로우**:
```
비밀번호 입력 → 대기 화면
  → "3위 공개" 버튼 → 3위 팀 카드 등장
  → "2위 공개" 버튼 → 2위 팀 카드 등장
  → "1위 공개" 버튼 → 1위 팀 카드 + 특별 이펙트
```

- 팀명, 프로젝트명, 소속사만 표시 (점수 미표시)
- 메달 아이콘 + shine 애니메이션
- 3위 → 2위 → 1위 순으로 위에서 아래로 쌓임

---

### 3-6. 관리자 집계 (`/admin`)

**대상**: 관리자
**인증**: 관리자 비밀번호 (Firestore 저장, 기본값: ai1234)

**화면 요소**:
- 요약 카드: 사전심사, 인기투표, 심사위원 현황, 예측 참여수, 참가팀
- **심사위원 관리**: 추가/삭제 (동적)
- **가중치 설정**: 심사 : 인기 : 사전심사 비율 (기본 50:30:20)
- 종합 순위 바 차트
- 심사위원별 상세 점수 테이블
- 사전심사 / 인기투표 / 예측 상세 목록
- **1위 확정** 버튼 (추첨용)
- 데이터 초기화

---

### 3-7. 추첨 화면 (`/draw`)

**대상**: 관리자 (대형 스크린 송출용)
**인증**: 관리자 비밀번호
**액센트 색상**: Emerald `#10b981`

**전제**: 관리자 페이지에서 1위 팀이 확정된 상태

**플로우**:
```
비밀번호 입력 → 1위 팀 표시 + 맞힌 사람 수 표시
  → "추첨 시작" 버튼
  → 슬롯머신 애니메이션 (3초)
  → 당첨자 1명 공개
  → "한 명 더 추첨" 버튼 (반복 가능, 중복 방지)
```

## 4. 점수 산출 공식

```
최종점수 = (심사위원 가중평균 × 심사비율)
         + (인기투표 환산 × 인기비율)
         + (사전심사 환산 × 사전비율)

* 기본 비율: 심사 50% : 인기 30% : 사전 20% (관리자 조정 가능)

심사위원 가중평균:
  = Σ(항목별 전체 심사위원 평균 × 항목 가중치)
  * 항목 가중치 기본: 혁신25 / 완성25 / 영향30 / 발표20

인기투표 환산:
  = (해당 팀 득표수 / 전체 투표수) × 10

사전심사 환산:
  = (해당 팀 득표수 / 전체 사전심사 투표수) × 10
```

## 5. 데이터 모델 (Firestore 컬렉션)

| 컬렉션 | Document ID | 필드 |
|--------|-------------|------|
| `settings` | 설정키 (예: `admin_password`) | `value` |
| `judges` | 심사위원 이름 | `created_at` |
| `public_votes` | 사번 (예: `A000001`) | `voter_name`, `team_id`, `voted_at` |
| `preliminary_votes` | 사번 | `voter_name`, `team_id`, `voted_at` |
| `judge_scores` | `{심사위원}__{팀ID}` | `judge_name`, `team_id`, `innovation`, `completeness`, `impact`, `presentation`, `submitted_at` |
| `predictions` | 사번 | `voter_name`, `predicted_team_id`, `predicted_at` |
| `draw_winners` | 사번 | `voter_name`, `drawn_at` |

- Document ID를 사번/이름으로 사용하여 자연스러운 UNIQUE 제약 구현
- 서버 없이 클라이언트에서 Firestore 직접 호출

## 6. 사번 형식

- **패턴**: 영문 1자리 + 숫자 6자리
- **예시**: `A000000`, `B123456`
- **정규식**: `/^[A-Za-z]\d{6}$/`

## 10. 운영 시나리오

```
[사전] 관리자 → /admin 접속 → 심사위원 등록 → 가중치 설정
  ↓
[사전심사] 직원들 → /preliminary 접속 → 사전 투표
  ↓
[본선 당일 - 발표 중]
  직원들 → /vote (인기투표) + /predict (1등 예측)
  심사위원 → /judge (항목별 평가)
  ↓
[발표 종료 후]
  관리자 → /admin → 집계 확인 → 1위 확정
  ↓
[시상식]
  관리자 → /ceremony → 대형 스크린 → 3위→2위→1위 순차 공개
  ↓
[추첨]
  관리자 → /draw → 대형 스크린 → 1위 맞힌 사람 중 랜덤 추첨
```
