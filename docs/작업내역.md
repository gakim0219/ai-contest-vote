## Firestore 컬렉션 구조

| 컬렉션 | Doc ID 규칙 | 주요 필드 |
|--------|-------------|-----------|
| `settings` | 설정키 (예: `admin_password`, `judge_weight`) | `value` |
| `judges` | 심사위원 이름 (예: `홍길동`) | `created_at` |
| `public_votes` | 사번 (예: `A000001`) | `voter_name`, `team_id`, `voted_at` |
| `preliminary_votes` | 사번 | `voter_name`, `team_id`, `voted_at` |
| `judge_scores` | `{심사위원}__{팀ID}` (예: `홍길동__1`) | `judge_name`, `team_id`, `innovation`, `completeness`, `impact`, `presentation`, `submitted_at` |
| `predictions` | 사번 | `voter_name`, `predicted_team_id`, `predicted_at` |
| `draw_winners` | 사번 | `voter_name`, `drawn_at` |

## 작업 이력

### v1 — 초기 구현 (로컬 버전)
- Express.js + better-sqlite3 백엔드
- React + Vite 프론트엔드
- 6개 페이지: /vote, /judge, /predict, /ceremony, /admin, /draw
- 로컬 SQLite DB

### v2 — 기능 추가
- 사번 형식 변경: 6~10자리 숫자 → `A000000` (영문1+숫자6)
- 관리자 비밀번호 변경: `대회관리2026` → `ai1234`

### v3 — 시상/심사/사전심사 개선
- **시상 화면 수정**: 3위→2위→1위 순서 공개 + 점수 표시 제거
- **심사위원 동적 관리**: .env 하드코딩 → 관리자 페이지에서 추가/삭제
- **사전심사 투표 추가**: `/preliminary` 페이지 신설
- **가중치 3개**: 심사:인기:사전심사 (기본 50:30:20)

### v4 — Firebase 마이그레이션
- SQLite → Firestore 전환
- Express 서버 → Firebase Cloud Functions (Blaze 필요)
- GitHub 레포 생성 및 푸시

### v5 — Spark 플랜 대응 (현재)
- Cloud Functions 제거
- 클라이언트에서 Firestore JS SDK 직접 호출
- Firebase Hosting (무료) 배포 완료
- 서버 없는 풀 클라이언트 아키텍처

## 배포 방법

```bash
# 1. 의존성 설치
cd client && npm install

# 2. 빌드
npm run build

# 3. Firebase 배포
cd .. && firebase deploy --only hosting,firestore:rules
```

## 초기 설정 가이드

1. https://ai-contest-vote.web.app/admin 접속
2. 비밀번호 `ai1234` 입력
3. **심사위원 등록**: 심사위원 이름 입력 후 "추가"
4. **가중치 설정**: 심사:인기:사전 비율 조정 후 "저장"
5. 각 페이지 QR코드 생성하여 현장 배포

## 주의사항

- Firestore Spark(무료) 플랜 일일 한도: 읽기 50,000 / 쓰기 20,000 / 삭제 20,000
- 경진대회 규모(~100명)에서는 무료 한도 내 충분히 운영 가능
- 관리자 비밀번호는 클라이언트 사이드 체크 (내부 행사용 수준의 보안)
- Firestore 보안 규칙은 현재 open 상태 — 필요 시 강화 가능
