# HD현대 AI Agent Challenge 투표 시스템 사용 가이드

## 개요

AI Agent Challenge 본선 평가를 위한 투표·심사·시상 시스템입니다.
사전심사, 현장 인기투표, 심사위원 평가, 1등 예측 게임, 시상식, 추첨까지 전체 행사를 지원합니다.

## 페이지별 사용 방법

### 1. 사전 심사 투표 — `/preliminary`

> 행사 전 사전심사 위원들이 가장 우수한 팀을 선택합니다.

**사용 대상:** 사전심사 위원

**사용 순서:**
1. 사번(영문1자리 + 숫자6자리, 예: `A000000`)과 이름을 입력합니다.
2. 5개 팀 중 가장 우수하다고 생각하는 팀을 선택합니다.
3. 투표를 제출합니다.

**참고:**
- 1인 1회만 투표 가능합니다.
- 이미 투표한 경우 이전 투표 내용이 표시됩니다.

---

### 2. 1등을 맞혀라! (예측 투표) — `/predict`

> 최종 1등 팀을 예측하는 이벤트입니다. 정답자는 추첨 대상이 됩니다.

**사용 대상:** 현장 참석자 전원

**사용 순서:**
1. 사번과 이름을 입력합니다.
2. 최종 1등이 될 것으로 예상하는 팀을 선택합니다.
3. 예측을 제출합니다.

**참고:**
- 1인 1회만 참여 가능합니다.
- 예측이 맞으면 추첨(행운의 추첨)에 자동으로 참여됩니다.

---

### 3. 심사위원 평가 — `/judge`

> 등록된 심사위원이 각 팀을 4가지 항목으로 평가합니다.

**사용 대상:** 관리자가 사전 등록한 심사위원

**사용 순서:**
1. 심사위원 이름을 입력하여 인증합니다.
2. 상단 탭에서 팀을 선택합니다.
3. 4가지 항목에 대해 1~5점(별점)을 매깁니다.
   - **혁신성** (가중치 25%)
   - **완성도** (가중치 25%)
   - **업무 영향도** (가중치 30%)
   - **발표력** (가중치 20%)
4. 모든 팀의 평가를 완료한 후 제출합니다.

**참고:**
- 관리자 페이지에서 사전에 심사위원을 등록해야 합니다.
- 이전에 제출한 점수가 있으면 자동으로 불러옵니다.

---

### 4. 현장 인기투표 — `/vote`

> 현장 참석자들이 가장 마음에 드는 팀에 투표합니다.

**사용 대상:** 현장 참석자 전원

**사용 순서:**
1. 사번과 이름을 입력합니다.
2. 5개 팀 중 하나를 선택합니다.
3. 투표를 제출합니다.

**참고:**
- 1인 1회만 투표 가능합니다.
- 인기투표 참여자는 현장 인기투표 랜덤 추첨 대상이 됩니다.

---

### 5. 관리자 페이지 — `/admin`

> 전체 투표 데이터 집계, 설정, 심사위원 관리를 수행합니다.

**사용 대상:** 관리자

**인증:** 관리자 비밀번호 (기본값: `ai1234`)

**주요 기능:**

| 기능 | 설명 |
|------|------|
| 현황 요약 | 사전심사·인기투표·심사위원·예측 참여 현황 |
| 심사위원 관리 | 심사위원 추가/삭제 |
| 가중치 설정 | 심사위원(50%), 인기투표(30%), 사전심사(20%) 비율 조정 |
| 최종 순위 | 가중치 적용 종합 순위 확인 |
| 상세 데이터 | 팀별 심사 점수, 개별 투표 내역 조회 |
| 1위 팀 확정 | 추첨을 위해 최종 1위 팀을 확정 |
| 데이터 초기화 | 모든 투표·평가 데이터 초기화 |

---

### 6. 시상식 — `/ceremony`

> 최종 1, 2, 3등을 순차적으로 공개합니다.

**사용 대상:** 관리자 (화면 공유용)

**인증:** 관리자 비밀번호

**사용 순서:**
1. 관리자 비밀번호를 입력합니다.
2. 버튼을 순서대로 눌러 결과를 공개합니다.
   - 첫 번째 클릭 → 🥉 **3등** 공개
   - 두 번째 클릭 → 🥈 **2등** 공개
   - 세 번째 클릭 → 🥇 **1등** 공개
3. 각 팀의 이름, 프로젝트명, 소속 회사가 표시됩니다.

---

### 7. 행운의 추첨 (1등 예측 추첨) — `/draw`

> 1등을 정확히 예측한 사람들 중 랜덤으로 당첨자를 뽑습니다.

**사용 대상:** 관리자 (화면 공유용)

**인증:** 관리자 비밀번호

**전제 조건:** 관리자 페이지에서 **1위 팀을 먼저 확정**해야 합니다.

**사용 순서:**
1. 관리자 비밀번호를 입력합니다.
2. 1위 팀 정보와 정답자 수가 표시됩니다.
3. `🎰 추첨 시작` 버튼을 클릭합니다.
4. 3초간 슬롯 머신 애니메이션이 재생됩니다.
5. 당첨자가 공개됩니다.
6. 추가 추첨이 필요하면 `🎰 한 명 더 추첨` 버튼을 클릭합니다.

**참고:**
- 이미 당첨된 사람은 다시 뽑히지 않습니다.
- 정답자가 모두 당첨되면 추첨이 종료됩니다.

---

### 8. 현장 인기투표 랜덤 추첨 — `/random-vote`

> 현장 인기투표에 참여한 사람들 중 랜덤으로 당첨자를 뽑습니다.

**사용 대상:** 관리자 (화면 공유용)

**인증:** 관리자 비밀번호

**사용 순서:**
1. 관리자 비밀번호를 입력합니다.
2. 전체 참여자 수와 추첨 가능 인원이 표시됩니다.
3. `🎲 추첨 시작` 버튼을 클릭합니다.
4. 3초간 슬롯 머신 애니메이션이 재생됩니다.
5. 당첨자가 공개됩니다.
6. 추가 추첨이 필요하면 `🎲 한 명 더 추첨` 버튼을 클릭합니다.

**참고:**
- 이미 당첨된 사람은 다시 뽑히지 않습니다.
- 인기투표에 참여한 모든 사람이 추첨 대상입니다 (팀 무관).

## 행사 진행 순서 (권장)

```
① 사전 준비
   └─ 관리자: 심사위원 등록, 가중치 설정 (/admin)

② 사전 심사
   └─ 사전심사 위원: 사전 심사 투표 (/preliminary)

③ 행사 시작
   ├─ 참석자: 1등을 맞혀라! 예측 투표 (/predict)
   └─ 심사위원: 심사위원 평가 (/judge)

④ 발표 후
   └─ 참석자: 현장 인기투표 (/vote)

⑤ 결과 집계
   └─ 관리자: 결과 확인 및 1위 팀 확정 (/admin)

⑥ 시상식
   └─ 관리자: 3등 → 2등 → 1등 순차 발표 (/ceremony)

⑦ 추첨 이벤트
   ├─ 관리자: 1등 예측 맞힌 사람 추첨 (/draw)
   └─ 관리자: 현장 인기투표 참여자 추첨 (/random-vote)
```

---

## 점수 계산 방식

최종 점수 = **심사위원 점수 × 50%** + **인기투표 점수 × 30%** + **사전심사 점수 × 20%**

| 항목 | 산출 방식 | 기본 가중치 |
|------|----------|-----------|
| 심사위원 점수 | 4개 항목 가중 평균 (1~10점) | 50% |
| 인기투표 점수 | (해당 팀 득표수 / 전체 투표수) × 10 | 30% |
| 사전심사 점수 | (해당 팀 득표수 / 전체 투표수) × 10 | 20% |

**심사위원 4개 항목 가중치:**
- 혁신성 25% / 완성도 25% / 업무 영향도 30% / 발표력 20%

> 가중치는 관리자 페이지에서 변경할 수 있습니다.
